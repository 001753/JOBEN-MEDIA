## 0. Ringkasan Revisi (v1.0 → v1.1)

Revisi ini **hanya** mengubah aspek lingkungan/infrastruktur. Seluruh scope produk, persona, user story, dan kebutuhan fungsional non-infrastruktur **tidak berubah**.

| Aspek | v1.0 (sebelumnya) | v1.1 (revisi ini) |
|---|---|---|
| Hosting utama | VPS milik sendiri | **cPanel / shared hosting**, dengan fondasi siap migrasi ke **cloud server / Google Cloud** |
| Database | PostgreSQL Neon/Supabase | **PostgreSQL bawaan cPanel** (koneksi localhost) |
| Domain | domain-utama.com (placeholder) | **news.jobenapp.cloud** (frontend) & **cms.news.jobenapp.cloud** (CMS/API) |
| Proses server | PM2 + Nginx (root access VPS) | **cPanel "Setup Node.js App" (Passenger)** — tanpa root access |
| Media storage | Cloudflare R2 | Tidak berubah — Cloudflare R2 (S3-compatible, independen dari pilihan hosting) |
| Kesiapan scale-up | — | Ditambahkan: pondasi migrasi ke Google Cloud (Dockerfile, dokumentasi env var, panduan migrasi DB) disiapkan sejak awal tanpa menghambat go-live di cPanel |

> **Asumsi nama subdomain:** `cms.news.jobenapp.cloud` digunakan untuk Strapi (CMS/API). Jika Anda menginginkan nama lain (mis. `api.news.jobenapp.cloud`), sesuaikan konsisten di seluruh dokumen ini.

## 1. Ringkasan Eksekutif

Membangun portal media berita online (sejenis detik.com, CNBC Indonesia, CNN Indonesia) dengan sistem redaksi formal (>5 orang), menggunakan arsitektur headless CMS. Target monetisasi utama adalah Google AdSense, dengan fondasi teknis yang mendukung SEO tinggi, kecepatan loading, dan skalabilitas untuk traffic besar.

### 1.1 Tujuan Produk
- Menyediakan platform publikasi berita yang cepat, SEO-friendly, dan mudah dikelola tim redaksi
- Mencapai kelayakan approval Google AdSense
- Membangun fondasi yang bisa scale ke traffic tinggi tanpa rombak ulang arsitektur — dimulai dari shared hosting (biaya rendah) dengan jalur migrasi yang sudah disiapkan ke cloud server/Google Cloud saat traffic tumbuh

### 1.2 Target Pengguna
| Tipe Pengguna | Kebutuhan |
|---|---|
| Pembaca (publik) | Akses berita cepat, mudah dibaca di mobile, navigasi jelas |
| Wartawan/Penulis | Input & edit draft berita dengan mudah |
| Editor/Redaktur | Review, approve, publish, kelola breaking news |
| Super Admin | Kelola user, role, kategori, pengaturan situs |
| Pengiklan (via AdSense) | Slot iklan yang tidak mengganggu UX tapi visible |

## 3. Persona & User Stories

### 3.1 Wartawan/Penulis
- Sebagai wartawan, saya ingin menulis draft artikel dengan editor teks kaya (rich text) agar bisa menyisipkan gambar dan format dengan mudah.
- Sebagai wartawan, saya ingin submit artikel untuk direview, bukan langsung publish, agar terjaga kontrol kualitas.

### 3.2 Editor/Redaktur
- Sebagai editor, saya ingin melihat daftar artikel berstatus "menunggu review" agar bisa cepat approve/tolak.
- Sebagai editor, saya ingin menandai artikel sebagai "Breaking News" agar tampil di posisi utama homepage.

### 3.3 Pembaca
- Sebagai pembaca, saya ingin homepage memuat cepat (<3 detik) di koneksi mobile agar tidak kabur ke situs lain.
- Sebagai pembaca, saya ingin mencari berita berdasarkan kata kunci atau kategori.
- Sebagai pembaca, saya ingin membagikan artikel ke media sosial dengan mudah.

### 3.4 Super Admin
- Sebagai admin, saya ingin mengatur siapa saja yang punya akses publish agar konten terkontrol.
- Sebagai admin, saya ingin melihat statistik artikel (views) untuk evaluasi performa konten.

## 6. Arsitektur Teknis

### 6.1 Arsitektur Saat Ini — cPanel / Shared Hosting

```
Cloudflare (DNS + CDN + SSL Proxy)
        │
        ├── news.jobenapp.cloud ───────► cPanel Node.js App #1: Next.js (SSR/ISR)
        │
        └── cms.news.jobenapp.cloud ───► cPanel Node.js App #2: Strapi (CMS/API)
                                                  │
                                                  ├──► PostgreSQL bawaan cPanel
                                                  │     (koneksi localhost, tanpa SSL)
                                                  │
                                                  └──► Cloudflare R2 (media gambar/video,
                                                        S3-compatible)
```

**Stack:**
- Hosting: **cPanel Shared Hosting**, fitur **"Setup Node.js App" (Passenger)** untuk menjalankan Node.js tanpa akses root
- CMS: Strapi (role-based, headless) — dijalankan sebagai Node.js App di subdomain `cms.news.jobenapp.cloud`
- Frontend: Next.js (App Router, ISR) — dijalankan sebagai Node.js App di domain utama `news.jobenapp.cloud`
- Database: **PostgreSQL bawaan cPanel** (fitur "PostgreSQL Databases"), koneksi `localhost` karena satu server dengan Strapi
- Media Storage: Cloudflare R2 (tidak berubah, tidak terpengaruh pilihan hosting)
- CDN/DNS: Cloudflare
- Domain: `news.jobenapp.cloud` (frontend) & `cms.news.jobenapp.cloud` (CMS/API)
- Monetisasi: Google AdSense

### 6.2 Jalur Migrasi ke Cloud Server / Google Cloud (Fondasi)

Disiapkan sejak awal agar bisa dieksekusi kapan saja tanpa rombak arsitektur:

```
cPanel Node.js App (Strapi + Next.js)
        │  dikemas ulang via Dockerfile yang sudah disiapkan
        ▼
Google Cloud
        ├── Cloud Run / Compute Engine ── menjalankan container Strapi & Next.js
        ├── Cloud SQL for PostgreSQL ──── hasil migrasi via pg_dump/pg_restore
        └── Cloudflare R2 tetap dipakai (S3-compatible, tidak perlu migrasi storage)

DNS & CDN tetap Cloudflare — saat migrasi cukup ubah A/CNAME record dari IP
shared hosting ke endpoint Cloud Run/Compute Engine (gunakan TTL rendah
sebelum migrasi untuk meminimalkan downtime).
```

Prinsip desain agar migrasi ini mulus:
- Semua kredensial & konfigurasi lewat **environment variable** (tidak ada hardcode path/host khusus cPanel di kode aplikasi)
- Media selalu di Cloudflare R2 (S3-compatible) — tidak pernah disimpan di local disk server, sehingga tidak ada beban migrasi storage
- Dockerfile untuk Strapi & Next.js disiapkan sejak Fase 1 (lihat prompt 10.7), meskipun tidak dijalankan di cPanel

## 8. Roadmap & Prioritas

1. Setup akun & tools: domain `news.jobenapp.cloud` (sudah ada), akun hosting cPanel, buat subdomain `cms.news.jobenapp.cloud`, aktifkan Cloudflare (DNS + CDN), siapkan GitHub repo
2. Tentukan struktur kategori berita & editorial guideline (SOP redaksi)
3. Rekrut/briefing tim redaksi (role: siapa jadi editor, siapa penulis)

### 🟡 Fase 1 — Fondasi Teknis (Minggu 2-4)
4. Buat database **PostgreSQL via cPanel** (menu "PostgreSQL Databases"): buat database, buat user, assign privilege ALL — catat nama database & user (format umum: `cpaneluser_namadb`)
5. Install & konfigurasi Strapi (development di Replit/local dulu boleh, deployment ke cPanel dilakukan di Fase 4)
6. Buat content-type: Article, Category, Tag, Author, Page
7. Setup role & permission (Super Admin, Editor, Penulis, Kontributor)
8. Setup Cloudflare R2 + hubungkan plugin upload provider ke Strapi
9. Testing alur upload gambar/video dari Strapi ke R2

### 🟤 Fase Paralel — Fondasi Migrasi Cloud (opsional, berjalan bersamaan, tidak menunda go-live)
9a. Siapkan Dockerfile untuk Strapi & Next.js (disimpan di repo, tidak dijalankan di cPanel)
9b. Dokumentasikan seluruh environment variable dalam `.env.example` sebagai sumber tunggal
9c. Susun `MIGRATION.md`: langkah pg_dump/pg_restore ke Cloud SQL dan deploy container ke Cloud Run/Compute Engine

### 🟠 Fase 2 — Frontend Dasar (Minggu 4-6)
10. Setup project Next.js, hubungkan API ke Strapi
11. Bangun homepage (headline, breaking news, kategori)
12. Bangun halaman detail artikel
13. Bangun halaman kategori & search
14. Implementasi desain responsif (mobile-first)

### 🔵 Fase 3 — SEO & Kepatuhan AdSense (Minggu 6-7)
15. Implementasi sitemap.xml otomatis
16. Tambahkan schema markup NewsArticle (JSON-LD)
17. Tambahkan Open Graph meta tag
18. Buat halaman wajib: About, Contact, Privacy Policy, Redaksi
19. Setup Google Search Console + submit sitemap (`https://news.jobenapp.cloud/sitemap.xml`)

### 🟣 Fase 4 — Deployment & Konten (Minggu 7-9)
20. Buat **2 Node.js App terpisah** di cPanel via "Setup Node.js App":
    - App Strapi → application root project backend, application URL `cms.news.jobenapp.cloud`
    - App Next.js → application root project frontend, application URL `news.jobenapp.cloud`
21. Jalankan `npm install` & build production (`strapi build` / `next build`) lewat interface cPanel atau Terminal SSH, lalu restart masing-masing app
22. Arahkan DNS via Cloudflare (A record ke IP shared hosting untuk kedua host, proxy aktif) dan aktifkan SSL (AutoSSL cPanel atau Cloudflare Universal SSL)
23. Redaksi mulai isi konten (target 20-30 artikel awal sebelum daftar AdSense)
24. QA menyeluruh: cek kecepatan, mobile, broken link, serta pastikan Node.js App tidak idle/sleep (lihat mitigasi di Bagian 9)

### ⚪ Fase 5 — Monetisasi & Go-Live (Minggu 9-10)
25. Daftar Google AdSense
26. Pasang slot iklan (header, in-article, sidebar) setelah approve
27. Soft launch — pantau traffic, uptime, error log, dan **resource usage cPanel** (CPU/RAM/proses)
28. Public launch / promosi

### Roadmap Lanjutan (Fase 2 Produk — setelah stabil)

## 9. Risiko & Mitigasi

| Risiko | Dampak | Mitigasi |
|---|---|---|
| Traffic melonjak tiba-tiba (berita viral) menyebabkan server lambat/down | Tinggi | Gunakan CDN Cloudflare, ISR di Next.js untuk kurangi beban server asal; jika kapasitas shared hosting cPanel terlampaui, eksekusi migrasi ke Google Cloud (pondasi sudah disiapkan di Bagian 6.2 & 10.7) |
| **Keterbatasan resource shared hosting cPanel** (kuota CPU/RAM/jumlah proses per akun) menyebabkan aplikasi lambat atau di-suspend hosting provider | Tinggi | Pantau pemakaian resource secara berkala di cPanel; upgrade paket hosting, atau jalankan migrasi ke Google Cloud sesuai fondasi yang sudah disiapkan |
| **Proses Node.js (Passenger) idle/berhenti** saat tidak ada trafik, menyebabkan cold start atau ISR tertunda | Sedang | Setup cron job (via cPanel "Cron Jobs") yang melakukan ping berkala (mis. tiap 10 menit) ke endpoint frontend & API agar proses tetap aktif |
| Konten dianggap duplikat/plagiat saat review AdSense | Tinggi | SOP redaksi wajib artikel orisinal, tidak copy-paste dari media lain |
| Tim redaksi belum familiar dengan Strapi | Sedang | Buat panduan/SOP internal + sesi training singkat |
| Video besar membebani storage R2 | Sedang | Kompres video sebelum upload, evaluasi Cloudflare Stream jika volume naik |
| Approval AdSense ditolak karena halaman legal belum lengkap | Sedang | Pastikan Privacy Policy, About, Contact sudah live sebelum daftar |

### 10.1 Prompt — Setup Strapi CMS (Fase 1)

```
Saya ingin membangun backend CMS untuk portal berita menggunakan Strapi (headless CMS).

Buatkan project Strapi baru dengan konfigurasi berikut:

   a. Article

      - slug (UID, based on title)
      - content (Rich Text)
      - excerpt (Text, short summary untuk meta description)
      - cover_image (Media, single, gambar)
      - video_url (Text, opsional untuk embed video)
      - category (Relation, many-to-one ke Category)
      - tags (Relation, many-to-many ke Tag)
      - author (Relation, many-to-one ke Author)
      - status (Enumeration: draft, review, published)
      - is_breaking_news (Boolean, default false)
      - views_count (Number, default 0)
      - published_at (Date)

   b. Category
      - name (Text, required)
      - slug (UID)
      - description (Text)

   c. Tag
      - name (Text, required)
      - slug (UID)

   d. Author
      - name (Text, required)
      - photo (Media, single)
      - bio (Text)
      - role (Enumeration: penulis, editor, admin)

   e. Page (untuk halaman statis: About, Privacy Policy, Contact, Redaksi)
      - title (Text, required)
      - slug (UID)
      - content (Rich Text)

3. Setup Role & Permission (di menu Users & Permissions Plugin):
   - Super Admin: akses penuh ke semua content-type
   - Editor: bisa create, update, publish/unpublish Article; tidak bisa hapus user
   - Penulis: hanya bisa create dan update Article miliknya sendiri, TIDAK bisa mengubah

4. Aktifkan REST API dan GraphQL API untuk semua content-type di atas agar bisa
   diakses dari frontend Next.js di domain news.jobenapp.cloud.

5. Konfigurasi CORS di config/middlewares.js agar API bisa diakses dari
   https://news.jobenapp.cloud (frontend production) dan http://localhost:3000

   startup terpisah (mis. server.js) di root project yang menjalankan Strapi
   dan memastikan Strapi listen di PORT dari process.env.PORT (bukan port
   default 1337 yang hardcode, karena cPanel akan menetapkan port sendiri
   secara otomatis). Jelaskan file mana yang harus saya set sebagai
   "Application startup file" di panel cPanel.

Setelah selesai, jelaskan juga cara saya generate API Token untuk akses read-only
dari frontend Next.js.
```

### 10.2 Prompt — Integrasi Cloudflare R2 sebagai Storage Media (Fase 1)

```
Project Strapi saya sudah berjalan. Sekarang saya ingin mengganti default upload
provider (local storage) menjadi Cloudflare R2, karena saya akan deploy ini ke
production (cPanel shared hosting) dan tidak ingin media tersimpan di local disk
server — selain lebih hemat kuota disk hosting, ini juga membuat media tidak
perlu ikut dimigrasi jika suatu saat pindah ke server/cloud lain.

Tolong:
1. Install provider upload yang kompatibel dengan Cloudflare R2 (R2 mendukung
   S3-compatible API, jadi gunakan provider AWS S3 seperti
   @strapi/provider-upload-aws-s3, dikonfigurasi untuk endpoint R2)
2. Buatkan file konfigurasi config/plugins.js dengan environment variable berikut
   yang akan saya isi manual di panel cPanel: R2_ACCESS_KEY_ID, R2_SECRET_ACCESS_KEY,
   R2_ENDPOINT, R2_BUCKET_NAME, R2_PUBLIC_URL
3. Pastikan konfigurasi mendukung upload gambar dan video dengan ukuran file
   hingga 100MB
4. Tambahkan validasi tipe file yang diizinkan: jpg, jpeg, png, webp, mp4, mov
5. Setelah dikonfigurasi, jelaskan cara testing upload dari admin panel Strapi
   (https://cms.news.jobenapp.cloud/admin) untuk memastikan file benar-benar
   tersimpan di R2, bukan local disk.
```

## 11. Catatan Tambahan

- Prioritas "Must Have" pada Bagian 4 adalah syarat minimum agar situs bisa go-live dan layak didaftarkan ke AdSense.
- Rekomendasi: jangan daftar AdSense sebelum minimal 20-30 artikel orisinal published dan seluruh halaman legal aktif.
